# 配置管理方案
项目：校园活动报名与签到管理系统

## 1 配置项识别
1. README.md
2. 项目章程
3. WBS 文档
4. 进度计划文档
5. 配置管理文档
6. 冲突过程记录
7. 实验总结报告

## 2 分支策略
- main：主分支，存放最终版本，受保护
- dev：开发汇总分支
- feature/*：文档/功能分支，如 feature-wbs、feature-conflict

## 3 提交规范
- 格式：姓名拼音: 简要说明
- 示例：liminghao: 完成WBS三级分解

## 4 合并规则
1. 个人分支 → dev 分支
2. 冲突解决后 → 提交 PR
3. 配置管理员审核 → 合并到 main
